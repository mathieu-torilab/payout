3.3 Reconciliation Flow

sequenceDiagram participant Admin as Admin participant RMS as Payout System (RMS) participant PES as Payout Execution Service participant Payoneer as Payoneer Admin->>RMS: Trigger reconciliation RMS->>PES: Request latest payout statuses PES->>Payoneer: GET payout statuses (batch) Payoneer-->>PES: Return statuses PES-->>RMS: Return results RMS->>RMS: Compare RMS records vs Payoneer results alt Mismatches found RMS-->>Admin: Display mismatches (table with discrepancies) Admin->>RMS: Review and resolve (manual override if needed) RMS->>RMS: Log resolution in audit trail else All matched RMS-->>Admin: Reconciliation complete, no discrepancies end
